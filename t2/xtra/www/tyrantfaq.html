The Tyrant FAQ

Here you can find almost everything you ever wanted to know about Tyrant, the game of Heroic Adventure.

What is Tyrant?

Tyrant is a roguelike game in which you play a lone hero in a world of adventure and magic. You start your journeys near a small town, tasked with a mission to recover an ancient artifact known as The Emerald Sword. You will be richly rewarded if you suceed, but many dangers lie in your path.....

What are roguelike games?

A game called Rogue started it all. It had nothing but ANSI terminal graphics to wow its audience, but still managed to create an atmosphere of adventure like no game before.

Opinions differ as to what constitutes a Roguelike. But the fundamental features seem to be:

  • Single main character
  • Overhead map view
  • Turn-based rather than real time play
  • Rich set of features and controls
  • Character advancement through gaining items and skills
  • Randomly generated maps, items and creatures

    • The emphasis in Roguelike games is usually upon strategy, resource management, exploration and taking chances.

    A great source for info on Roguelike games in general is Darren Hebden's Roguelike News. Darren is also responsible for a lot of of the graphics tiles in Tyrant!

    What's with all these keyboard commands?

    You need them, honestly. It would take hours to get anywhere if you had to point and click with the mouse everytime you wanted to do something. Keys are much faster, and once you get into some serious adventuring they will begin to feel like second nature.

    Most of the keys are pretty intuitive. 'e' for eat, 't' for throw, 'k' for kick etc. If you need help, you can always press '?' in the main game screen to see a full list of commands.

    I'm using a high-resolution screen and the graphics are way to small!

    Well, in the latest version you can zoom in and out of the game area by pressing F2 and F3. But this doesn't seem to work perfectly on all browsers as yet.

    What's coming in future versions of Tyrant?

    Once the game engine is properly finished, Tyrant should start growing pretty fast. The current plan is to have a whole world generated with a healthy mix of chaos and fractals in which you can undertake a wide variety of quests.

    There will be a few major plotlines, but little restriction on what the player chooses to do. It will be possible to win in a number of different ways, giving plenty of scope for everyone to develop their distinctive strategies.

    It's a particular ambition of mine to build the ultimate roguelike city, teeming with quests, intrigue and AI personalities. And of course The Tyrant, exerting its malevolent power over it all.

    Can I download Tyrant?

    Sure thing. Just grab tyrant.zip. Once unpacked, you can play the game locally using your favourite browser, or alternatively via a Java runtime environment.

    Tyrant is cool, but why don't you make it multiplayer?

    Though it would be fairly easy to build a multiplayer roguelike engine, it would be pretty unplayable since you would have to wait for other players to make their moves. The step-time nature is an intrinsic feature of Tyrant that can't really be changed without destroying the whole essence of the gameplay.

    Having said that, if anyone can think of a good way to capture the roguelike atmosphere in a multiplayer setting, then I would love to hear it.

    What language is Tyrant written in?

    The game is written in Java version 1.1.

    The development environment used is KAWA version 3.21 by Tek-Tools.

    Can I have the source code?

    You can indeed. Check out the Developer Site on SorceForge. All the source code is available under the GNU Public License which says that you can copy the code, modify it and basically do what you like providing that you also release your changes under the GPL.

    Tyrant is an Open Source project, which means that anybody can get involved and contribute to the game. If you can code in Java, you are welcome to get involved in fixing bugs and making improvements in the game. If not, you can still contribute ideas, create graphics and help spot bugs. If you're interested let me know.

    I've found a bug!

    Congratulations. There are plenty for everyone. Best way to get it fixed is to report it on the Bug Reports
    page on the SourceForge development site.

    If you do post a bug report, please try to be very specific about what happens so that I can track it down quickly. If it's a problem with the game mechanics, try to include enough information so that I can repeat the bug. If it's a problem getting the game working, please let me know what Java/Browser/OS version you are using.
